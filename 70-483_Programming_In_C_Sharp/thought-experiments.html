1 Using Multiple Tasks

At the design stage of an application you will have to decide whether/how to use tasks and threads.

Here are some questions to consider:

  1. Given the difficulties in synchronization and maangement, is it worth the effort to implement applications using multiple tasks?
    • Breaking an application into separate tasks is good for more than just performance advantages.
    • Tasks provide a very useful level of abstaction.
    • Once communication between tasks has ben designed, they can be worked on independently.
    • Partitioning an application into tasks can be seen as an extension of component-based design.
  2. Is it still worth the effort to use multiple-tasks if you only have one processor in your computer?
    • If the time taken to process a task is determined by the speed of the CPU, the program is CPU bound.
    • If a system has one processor, a multi-threaded implementation whould not improve the rate of processing.
    • A program that is contunously interacting with a file system is IO bound, as it's dependent by the performance of the file system.
    • A multi-threaded implementation could perform processing at the times when one task is waiting for an IO operation to complete.
  3. What kind of applications benefit the most from the use of multi-tasking applications?
    • A CPU bound application will benefit most if the host system has more than one CPU.
    • An application with a mix of CPU bound and IO bound tasks benefits most as there will always be something for the processor to do.
  4. Are there situations when you really should not use multi-tasking?
    • The asynchronous nature of multi-threaded applications makes it bery hard to guardantee that a given operation will be completed within a specified time.
    • Threads can be given priority levels, but these are not hard and fast.
    • Applications with critical timing requirements should not be implemented using multiple threads.
  5. I need a background process that is going to compress a large number of data files. Should I used a task or a thread?
    • Tasks are easier to create and are good for background processes.
    • Threads can be used for both background and foreground processes.
    • In this case tasks are better because they're easier to create and never need to operate in the foreground.
  6. What is the difference between the WaitAll and WaitAny method when waiting for a large number of tasks to complete?
    • WaitAll will return when all tasks that are running have completed.
    • WaitAny will return when the first of the running tasks completes.
  7. What is the difference between a continuation task and a child task?
    • A continuation task is a task which starts while an existing task completes.
    • A child task is created by a parent task and executes independetly of the parent.
  8. What is the difference between the WaitAll and WhenAll methods?
    • WaitAll accepts a list of Task references as an argument and returns when all the tasks have been completed.
    • WaitAll does not return a result.
    • WhenAll accepts a list of Task refernces as an argument and returns a Task that delivers the results returned by those tasks.
    • WhenAll can be awaited, which allows it to be used in asynchronous code.
  9. What happens when a method call is awaited?
    • Methods that can be awaited return a reference to a Task.
    • When an await in an asychrnous method is performed, the asynchronous method returns immediately.
    • When the task returned by the awaited method completes the remained of the asynchronous method will complete.
    • A good use of this is in user interfaces; The buttons behaviour must return quickly so the program doesn't appear unresponsive. By using await, the button click handler can complete and return, while the action intiated by the button click runs in parallel.
  10. What is special about a concurrent collection?
    • A concurrent collection is one which has behaviours that can be used by mulitple processes at the same time.
    • Some concurrent collections, like ConcurrentDictionary, have additonal behaviours that allow processes to only perform updated to data stored in the collection when it's valid to do so.

2 Managing Multithreading

Multithreaded applications will need to give special consideration to the way that data is processed and the ordering of operations. They will also have to contain provision for process management.

Here are some questions to consider:

  1. Will program errors caused by a poor multithreading implementation always manifest themselves as faults in an application?
    • There is no guarantee that the conditons that might cause a race condition to cause a problem will arise.
    • Natural system delays may mean that events occur in an order where race conditions do not cause a problem.
    • If problems occur it is wise to first check if any changes have been made to the underlying system.
  2. Does the fact that the processor is suddenly at 100% loading indicate that two processes are stuck in a deadly embrace?
    • The simplest form of deadlock occurs when two tasks are waiting for locks to be released by each other.
    • Complex deadlocks can be spread over a large number of tasks.
    • When a deadlock occurs nothing is happening at all, so the application is taking no processor time at all.
  3. Could you make an object thread safe by enclosing the body code of all the methods in lock statement to make all the method actions actomic?
    • Making all method calls in an object atomic is very heavy handed and dangerous approach to acheiving thread safety.
    • It may make an application vulnerable to deadlocks and could remove any performance advantage that would be gained from multiple processors.
    • It does not protect against issues caused by the use of reference parameters to a method.
  4. If you're not sure about potential race conditons, is it best to add lock statements around critical sections of your code "just in case"?
    • Adding lock statements around crtiical pieces of code without thinking about what the code actually does is very dangerous.
    • It may lead to perfomance bottlenecks and deadlocks.
    • It is very difficult to add thread safety to a program once it is written.
    • The only sure-fire way to write thread safe code is to design it that way from the start.
  5. Should a task always generate an exception if it is cancelled?
    • A task can be cancelled "silently", in which case it will just end.
    • It can be made to generate an exception when cancelled.
    • To some, cancelling a task is considered an exceptional action and would therefore expect there to be some way of propagating this event through the system so it can be registered in some way.
  6. Could you make an application that automatically cancelled deadlocked processes?
    • A Monitor construction can be used to manage entry into atomic actions where it is possible for a task to retain control when it fails to get a lock and enters an atomic action.
    • A time-out can be established by providing a timeout value to TryEnter which tries to enter an atomic action.
    • Therefore, it is possible for a program to recognize that a given lock has been unavailable for over a specified amount of time.
    • A cancellation token can be used to try and stop a task that mihght have obtained the synchronization object.
    • This only works correctly if the task that is causing the deadlock is checking its cancellation token while it is waiting for its lock.
    • TL:DR; Yes it's possible, it just entails a lot of extra work. The best way to avoid deadlocks is to make sure they aren't present in the design of the system.

3 Program Flow

The C# language provides constructions that the programmer can use to manage the flow of program execution. Statements can be repeated a given number of times, until a specific state has been reached, or over a collection of items. Statements can bee xceuted conditionally using the if construction, or selected on the basic of a particular control variable in the case of a switch.

Here are some questions to consider:

  1. Is it necessary to have both the while and the do-while looping constuction?
    • The while construction will not execute any code if the control condition is false.
    • The do-while consutrction tests the control condition at the completion of the loop, so it always executed once.
  2. Is a break statement the only way of exiting a loop before it completes?
    • A break statement will exist the loop at that point.
    • A return statement will exist the method in which the loop is declared.
    • An exception being thrown will exit the loop in which it occurs.
  3. Can you identify a situation where you would use a for loop to iterate through a collection rather than a foreach loop?
    • A foreach construction provides a convenient way to express a need to enumerate each of the items in a collection.
    • Each enumerated item is only available on a read-only basis.
    • A for loop is needed if you want to change the value of an enumerated item.
    • The properties of enumerated items in a collection of reference types can be changed in a foreach loop.
  4. The & (and) logical operator and the | (or) logical operator have "short circuit" versions && (conditional-and) and || (conditional-or), which only evaluate elements until it can be determined whether or not a given expression is true or false. Why does the ^ (exclusive-or) not have a short circuit version?
    • There is no short circuit version of exclusive-or because it is not possible to determine whether an exclusive-or is true by examining a single operand.
    • An exclusive-or returns true if the two operands are different, and that determination cannot be made looking an just one operand.
  5. Is it true that each C# operator has a behaviour that is the same for every context in which it is used?
    • No, C# operators have different behaviors depending on the context in which they are used.
    • Integer and floating-point types are held in differnt ways in the memory of the computer.
    • When arithmetic operators are used between these types the compiler will select the correct operator to be used depending on the type of the operands in the expression.
  6. Can you always be certain of the precise sequence of operations when an expression is evaluated?
    • The C# compiler guaratees that expressions will be evaluated according to the rules of precedence and associability for the operators in an exception.
    • However, the precise sequence in which the operations are performed is not something that a programmer can make assumptions about.

4 Events and Callbacks

Programmers can use delegates to create applications from "loosely coupled" objects that communicate by message events. An object can subscribe to events published by another object which exposes a delegate.

A lambda expression can be used to create "anonymous functions" that exist as pure behaviours. Delegates can be made to refer to anonyoumous functions.

Here are some questions to consider:

  1. How does a delegate actually work?
    • A delegate is a type that can refer to a method in an object with a particular signature.
    • A delegate can also contain a list of method references.
    • A delegate is called in exactly the same way as a method it can refer to.
    • If the delegate contains a list of method references, each method is called in no particular order.
    • The base type of all delegate objects is the Delegate class.
  2. What happens if a delegate is assigned to a delegate?
    • Delegates can be assigned to other delegates, as long as they have the same signature.
    • Delegates can also be added to the subscribers of a multi-cast delegate.
  3. What happens if a delegate is assigned to itself?
    • A delegate can be assigned to itself, but this is not a sensible thing to do.
    • If the delegate is ever called the program will enter an infinite loop.
  4. Is there an upper limit for the number of subscribers that a publisher delegate can have?
    • There is no upper limit, over and above the capacity of the data structure in the Delegate class which holds references to subscribers.
  5. What does the => (lambda) operator actually do?
    • The lambda operator separates the items that go into the expression from the statement or statements block that implements the behaviour of the lambda expression.
  6. Can code in a lambda expression access data from the enclosing code?
    • Code in a lambda expression can access variables declared in code enclosing the lambda expression.
    • Code in a lambda expression is not executed when the lambda expression is declared, a reference to the lambda expression my be followed at a later time.
    • The C# compuler will generate code that maintains variables used by the code in the lambda expression even if they would normally be out of scope at the time the code in the lambda expression is performed.

5 Exceptions

Exceptions are an important part of application development. At the very least an application will have to handle exceptions that may be produced by .NET methods that it uses. For a large application you may have to design custom exception types and create an error handling strategy that manages how exceptions are propagted and managed. It is important to put exceptions in the correct perspective; any error that you would expect the system to deal with in the normal course of events should not be handled by the use of an exception, they should be reserved for situations where it is not meaningful for the application to continue.

Here are some questions to consider:

  1. Is a method receiving a date of "31st of February" something that should cause an exception.
    • It depends on the context of the action.
    • If the date is being entered by a user it is reasonable to display an error message and request that the date be re-entered.
    • If the date is being supplied by a method that is expected to create a transaction with this date, it is reasonable for that method to throw an exception if given an invalid date, as it wouldn't be meaningful for the method to generate anything.
    • Throwing an exception, rather than returning an error code or a null value, is a way of ensuring that the program will fail in a properly noticeable way and maximizes the chances of the mistake being detected.
    • From a design perspective, the need for exceptions in this situation would be completely removed if the method creating the trasaction was passed a DateTime structure. This can only contain a valid date.
  2. Should I make sure that all exceptions are always caught by my program?
    • The only thing worse than a program that fails, is one that fails silently.
    • You should catch all the exceptions that you know how to deal with.
    • Let the remaining ones pass through so they are either caught by layers above or propagate as errors which can be caught and dealt with.
  3. Can you return to the code after an exception has been caught?
    • There is no way to return to the statements after an exception has been thrown.
    • If a program wants to re-try code that may throw exceptions, this code must be placed in a loop.
  4. Why do we need the finally clause? Can't code to be run after the code in a try clause just be put straight after the end of the catch?
    • The finally clause is needed because code in the catch clause may return from a method or throw other exceptions.
    • In which case the program will not reach the statements following the catch.
    • The finally clause is guaranteed to be executed in all circumstances.
  5. Should my application always use custom exceptions?
    • The standard Exception type provides a great deal of flexibility.
    • It contains a Dictionary which can be filled with name-value pairs that describe an error.
    • Custom exception types can be useful if none of the existing exception types are suitable for the exception context.
    • Different catch clauses can be allocated to different exception types, they also make exceptions easier to manage.

1 Creating Types

There comes a point in application construction where the data storage for the application must be designed. At this point you need to map data items onto C# data types. Each data item will be mapped onto an object that may be manged by value or reference. State information can be mapped onto enumerated types for which yuo can set a range of allowed values.

Here are some questions to consider:

  1. You want to store information about the employees that work in my company. Is it a job for a value type or reference type?
    • If you want to handle different types of employee then a reference would be better.
    • Reference types will make it much easier to index and sort your employee records.
    • Structures are best suited to small value types, which you may want to make immutable.
    • Using structures to hold the data would be a good idea if the numbers of data items is very large.
  2. Can an object be referred to by more than one reference?
    • Yes, a single object can have mulitple references referring to it.
  3. What happens when an object no longer has any references to it?
    • It can be destroyed by the garbage collector as it has no more references to it.
    • At runtime the garbage collection process checks for objects that have no references and removes them.
  4. If you use a value type rather than a reference type, will your program still work correctly?
    • It is possible to create any program entirely using reference types of vale types.
    • The behaviour of assignments and the effects of changes to objects means the program may not behave as you expect.
  5. How can you make a data type immutable?
    • A data type can be made immutable by providing no means of changing the varaibles held inside the type.
    • The values of a data type should be set by the constructor and the type should only provide methods and properties that can be used to read these variables.
    • The variables in the type must all be made to be private.
  6. Are value types more efficient between than reference types?
    • Value types can be regarded as slightly more efficient to access than reference types.
    • To access a reference type a program mst follow the reference to the position in the heap where the object is stored.
    • A value type can be accessed directly and an array of value types is stored in a single block of storage that contains all of the values.
    • The performance difference will not be noticeable except in extreme circumstances where huge numbers of objects are being processed.
  7. What is the difference between the stack and the heap?
    • Space on the stack is allocated as a program runs.
    • Each time a block is entered, an area of stack space is reserved for value type variables taht are created when the method runs.
    • When the block exists, all of the stack space is recovered and the value type variables deleted.
    • Space on the heap is allocated when referemce types are created.
    • A reference type may have a lifetime that is longer than the block in which it is declared.
    • The heap has no particular structure, so there is no guarantee that successively craeted objectes will be in adjacent locations on the heap.
  8. Do you need to add a constructor method for every object that you create?
    • Most types require at least one constructor, so that your application can ensure that the object has a valid initial state and can feed information into the type.
    • The cosntruction of the objects in your system is something that should be planned at the time the system is designed.
  9. Is there ever any point in creating objects that cannot be constructed?
    • A class that only contains static memebers does not need to be constructed.
    • You find static classes like this that contain service methods.
  10. What does it mean when a member of a type is made static?
    • A static member of a type exists as part of the type, not a given instance of the type.
    • It is accessed via the name of the class.
    • Static memebers are great for storing things like limits and default values as these only need to be stored for an entire class, not for each object.
  11. Should you provide default values for all of the parameters to a method?
    • A program can provide default values for method parameters.
    • This can be useful, as it can simplify the use of the methods.
    • It provides a form of information hiding, in that users of the method may not know what the default values are.
    • I restrict default values to parameters that specify "optional" rather than "core" behaviours.
  12. Does using a lot of overriden methods slow a program down?
    • A method that is marked as virutal can be overriden in a class hierarchy.
    • When an overridden method is called on a reference the runtime system must search up the class hierarchy for the version of the method to use.
    • The deeper the class hierarchy the longer it will takes for this seach to find the method.
    • The extensive used of overridden methods in a class hierarchy may have an impact on program performance, but it is unlikely to be noticeable unless the class hierarchy is very deep.

2 Consuming Types

Once you have established types that your program will use, you will have to write code to manipulate the types. This frequently means that the program will have to convert values from one type to another, and sometiems a program will have to interact with external services which do not have type information that can be used by the C# compiler to perform static analysis to establish the correctness of an interaction.

Here are some questions to consider:

  1. If every variable in a program was maanged by reference, would you ever need to box a value?
    • Boxing is the name given to the action of converting a value type into a reference type.
    • It is performed when we need to work with a value as an object, perhaps to allow the value to be inserted in a data structure that is managed by reference.
    • A program will spend a lot of its time just manipuklating values, and the use of reference types will significantly slow down this process.
    • You can run a program using only interface types that provide the object implemtations of value types, but this would run a lot more slowly than one that used value types.
  2. Do you need to use a cast when your program is widening a type?
    • Widening is when a value is being moved from one type to another with a wide range of possible values.
    • E.g. from an integer to a double.
    • In this situation no information is lost, so casting is not required.
  3. Does casting between values slow a program down?
    • Casting between values types may be something that the CPU can perform very quickly.
    • However, if the casting process involves calling a type conversion method this will take longer.
  4. Can an object have multiple type conversion methods?
    • A type conversion method is added to an object to generate a value to be used when an object is cast into a particular type.
    • Each type conversion method is assigned a "target" type, and so it would be possible for an object to have a type conversion to multiple types.
    • In these situations it's recommended to use properties to return a value from a type, rather than using casting and type conversion methods.
  5. Could you write a program that used entirely dynamic types?
    • You can write a program that uses entirely dynamic types.
    • It should always compile since the compiler will not perform any static error checking.
    • However, it would probably fail quite frequently too, as mistakes that would normally produce compilation errors would not be spotted and would manifest themselves as run time errors.
    • Using dynamic types also makes developement more difficult as Dynamic types acquire their types at runtime and so the Intellisense feature of Visual Studio cannot provide any useful information about type a variable holds.
  6. Why do you need to interact with Component Object Model (COM) objects?
    • Many applications provide automatic features taht are accessed via COM interfaces.
    • This is also true of some of the lower level elements of the Windows operating system.
    • It is unlikely that you will have to interact with lots of COM objects during your programming career, but it is important to be able to work with them.

3 Enforce Encapsulation

Encapsulation is a powerful technqiue. It allows developers to hide the implementation of an object and control precisely how code outside the object will interact with it. It also means that develoerps can change the internal design of an object without affecting any of the users of that object.

Here are some questions to consider:

  1. Is it better to use a property rather than get and set methods for private data members in your classes?
    • There are 2 ways that an object can encapsulate data values: via porperties, or via getter and setter methods.
    • Properties are very easy ways for code to interact with encapsulated values, as they're assigned and read in the same way as public data values.
    • There is no difference in performance between properties and get/set methods.
    • The compiler generates get and set methods when properties are compiled.
    • However, get and set methods are slightly more flexible, in that a set method can perform validation on an input value and return a result which indicates whether an input value has been set correct.
    • The set behaviour can also perform validation, but the set behaviour cannot return a status value.
  2. Can structure contain private data members and properties?
    • There is no reason why a structure should not contain private members and properties.
    • Although most people consider encapsulation as something that is performed on classes, structures can be encapsulated objects.
    • An extreme form of encapsulation is the creation of immutable objects.
    • Immutable objects provide read-only properties and accessor methods to allow other programs to view the contents of the object.
  3. Does using properties slow down the execution of my program?
    • Properties will slightly slow down a program, in that a property behaviour will take longer to complete than just getting a value directly from an object.
    • The property access code will be inlined, rather than calling a function to perform the property behaviour the compiler will just output the code required.
    • This makes program slightly larger but also means that accessing properties in objects is almost as fast as accessing the values directly.
  4. Which provides a greater amount of access to a class memeber, protected or internal?
    • A protected member is visible in that class and in any classes thar are children of that class.
    • An internal member is visible to any class that is declared in the same assembly.
    • The internal item has the potential to be seen by more classes in an application.
  5. Are there any reasons not to use explicit implementation of methods in an object?
    • The only reason that an object would not use an explicit implementation of interface methods is if the programmer wants to "share" some methods between several internal interfaces.
    • This is not the kind of thing sensible programmers should do.

4 Create and Implement a Class Hierarchy

The starting point for a class hierarchy should not be the design of the base class of the heirarchy, but the creation of an interface that expresses the behaviours of this base class. Using an interface from the start of your design process allows a great deal of flexibility in that objects can be regarded in terms of their abilitie, not their particular type. You can still use a class hierarchy to reduce the amount of code that your have to write, by creating child classes that inherit methods from their parents and only override the one that need to provide behaviours specific to the child.F

Here are some questions to consider:

  1. Should you design your interfaces after you have designed your application?
    • The behaviours that are required by an object are best expressed in the form of an interface.
    • You might create a proof of concept application and then create interfaces based on the behaviours of the objects in that application.
    • You should make sure the interfaces are established right at the start of the development process.
  2. Can a C# interface contain anything other than a method signature?
    • They can also contain the definition of properties that should be created by any class that implements the interface.
  3. Can a C# interface be extended?
    • Yes, a C# interface can be extended.
  4. Are all the methods defined in an interface public?
    • Yes, all methods in an interface must be made public.
    • However, an interface can be made internal, which means that it is only visible within a given assembly.
  5. Does a class have to ipmlement all the methods in an interface that it implements?
    • No, a class must contain implementations of all the methods in an interface if we want to create instances of the class.
    • A class in a hierarchy could implement some methods in an interface and a child method of the class could implement the rest.
    • In this situation an instance of the base class could not be created. This design approach is not encouraged.
  6. Can a C# interface contain a constructor method?
    • No, interfaces do not contain constructors.
    • If you want to go down the route of having interface methods that construct instances you will have to put factor methods in the interface.
  7. Is it sensible for an object to implement more than one interface?
    • It can be.
    • For example, an Employee record that must be printed could expose an IEmployee interface and an IPrintable interface.
  8. Can only the base class of a hierarchy implement an interface?
    • No, interfaces can be implemented at any level of a class hierarchy.
    • If the base class implements the interface, this forces any children of the base class to implement the interfaces too, but child classes can implement them further.
  9. Does making methods virtual slow a program down?
    • Yes, very slightly.
    • A non-virtual method can be called directly, since there will only ever be one copy of that method.
    • When a virtual method is called the .NET runtime must search for the most appropriate method to call, starting with the type of the object and then looking up the class hierarchy to find the method in base classes.
  10. Does making methods virtual reduce the security of a program?
    • Yes, a method that is virtual can be overriden and replaced with one that has a different behaviour in the child.
    • You should only do this for methods which you have a definite need for the ability to override their behaviour.
  11. What is the difference between a C# interface and an abstract C# class?
    • An interface specifies a set of behaviours that an objet must provide so that it can be referred to by a reference of the interface type.
    • This provides a lot of flexibility, for example a list of interface references could contain any type of object, as long as that type implemeneted the interface.
    • An abstract type is more restrictive. It contains a set of behaviours that an object must provide, but the object can only then be regarded in terms of the abstract types, not in any other way.
  12. Should every class that you create be part of a class hierarchy (either as a base class or a child class)?
    • No, if there is no need for an object to be extended then it does not need to be part of a hierarchy.
  13. Can a method be both overridden and overloaded?
    • Yes it can.
    • However, if a method is going to be overridden then be wary of providing too many overloaded forms of the method in the base class.
    • This is because the child class will have to provide overriding implementations of all the overloaded versions, otehrwise the behaviour of the child objects may not be correct.
  14. Can you override a method in a structure type?
    • No, structures are value types and cannot be used in class hierarchies.
  15. Can a structure implement an interface?
    • Yes, although structures cannot be used in class hierarchies they can be declared as implementing interfaces.
    • A structure value will be boxed if it is to be accessed via an interface reference, but this will happen automatically.
  16. What does it mean when a class is sealed?
    • A class that has been sealed cannot be used as the base of a child class that extends it.
  17. Can any object implement the IEnumberable interface?
    • Yes, any object can implement IEnumerable.
    • However, it is only meaningful for items that contain or deal with enumerable collections to implement this interface.
  18. Does the yield keyword cause my program to stop?
    • No, it causes an enumerator to return the value specified by the yield return.

    • Not necessarily. The use of using means that the system can be absoloutely clear when an object is no longer required.
    • But, this will not inoke a garbage collection action to remove that object.

6 Manage the Object Life Cycle

The automatic garbage collection enjoyd by C# develoeprs is built on carefully written and highly developed foundations. However, the non-deterministic nature of garbage collection process (you can't always be sure precisely when an object will be removed from memory) means that programmers must exercise care when working with objects taht contain references to resources that are to be shared with other objects. The IDisposable interface provides a way that programmers can enforce a more managed approach to resource management.

Here are some questions to consider:

  1. If you use value types rather than reference types, will this stop the garbage collector from being called?
    • Value types are normally stored on the stack.
    • When a program exits from a block the variables on the stack are automatically removed.
    • This means creating and destroying types such as integer, float and double will not make any work for the garbage collector.
    • However, if the value type is a struct, and the struct contains reference types, it will create work for the garbage collector, as the objects inside the structure will need to be managed.
  2. Does creating and destroying arrays of value types make work for the garbage collector?
    • Arrays of value types are implemented as objects.
    • This means that such arrays wil be implemented on the heap and their removal from memory will involve the garbage collector.
  3. Does the garbage collector also collect garbage from the stack?
    • No, the stack grows and contracts as a program runs. It is not managed by the garbage collector.
  4. Can a finalizer be called before the Dispose method in an object?
    • Yes, it is the application developer's responsibility to make sure that the Dispose metod is called to request that the object release any resources that it is using.
    • If this does not happen, and the object is deleted from memory, the finalize method will be called first.
    • The dispose pattern has been designed to ensure that an object would release resources in a sensible way in this situation.
  5. Can the finalizer method in a class be overloaded?
    • No, it would not be meaningful for the finalizer method in a class does not accept any parameters, so overloading the method is not possible.
  6. Does a finalizer in a child class have to call the finalizer in a base class when it runs?
    • No, there is no need for a finalizer in a child class to make any class to the finalzier in a parent class.
    • Each class in a class hierarchy can have its own finalizer and each of them will be called in turn when the object is finalized.
  7. Is it possible for the Dispose method in an instance to be called more than once?
    • It is possible to call Dipose multiple times.
    • The dipose pattern uses a flag to ensure that this does not cause problems.
  8. What happens if an object finalizer takes a long time to run?
    • A long running finalizer will seriously impact on the performance of the garbage colletion process.
    • In an extreme situation this may result in the application reunning out of memory as the garbage collector cannot recover memory faster than it is being used.
  9. What happens if an object finalizer creates a new reference to the object being finalized?
    • It is bad practice for a finalizer to create a new reference to itself.
    • This results in the object not being delete from memory.
    • Remember that the garbage collector makes another pass through the heap after the finalizers have been called.
    • Any reference to it will be retained.
  10. Should you design your applications from the start to use the minimum possible memory?
    • Security should be designed into an application very carefully from the start.
    • However, in regard to performance and memory use, it is advised you don't do much mroe than avoid obviously stupid solutions that impact on these factors, and then focus on creating an application that is easy to understand and test.
    • If you subsequently discover that you have performance or memory issues, that is the point that you use your diaganostics tools to identify resource hungry elements of your program and then optimize those.
    • A lot of developers have spent a lot of time making programs smaller and quicker when there is no need to do this, particularly with modern hardware.

7 Manipulate Strings

Successive version of C# have added more and more string manipulation abilities to the language and its libraries. It is very unlikely that you will find yourself in a situation where you have to write some code to perform string processing. I strongly advise you to check out the abilities of the string type and the libraries that work with it before you start to write custom code to work with strings.

Here are some questions to consider:

  1. Can you hold the text of an entire book in a single string instance?
    • "War and Peace" is considered to be a long book of half a million words. If each word was 20 characters long, that'd equate to around 10 million characters long.
    • Strings are indexed using a 32-bit integer, which makes the theoretical limit on string size at around 2,000 million characters.
    • So, you could say that all novels will fit in a single string.
  2. A string object is immutable. Does this mean you can't assign a new value to a string variable once you have set it to a value?
    • The contents of a string object cannot be changed. However, a string variable can be made to refer to a different string instance.
  3. Are there any disadvantages to interning strings in a C# program?
    • Interning of strings takes place when the compiler notices the same string literal is assigned to multiple variables in the program source.
    • The compiler makes a single "interned" variable that contains the string literal and all the variables assigned that literal are made to refer to a single value.
    • At tun time this has the advantage that the program may be able to compare strings more quickly because two string variables will be equal if they both refer to the same interned object.
    • The only disadvantage of interning is that it slows down the compilation process.
  4. Is the StringBuilder type based on the string type?
    • The StringBuilder type is not based on the string type.
    • The StringBuilder will use one or more arrays of characters to build a string.
  5. Will using StringBuilder always make your programs go faster?
    • If a program does a lot of string addition to assemble an output string it may be that using StringBuilder to assemble the string will improve performance and perhaps reduce the number of intermediate objects that will be created.
  6. Can you store binary data in a StringWriter?
    • You can't store binary data in a StringWriter because a StringWriter implements the TextWriter interface, which can only accept strings of text.
  7. Does the Trim method change the contents of a string?
    • The Trim method is used to remove whitespace from an input string and returns a new string with the whitespace removed.
    • It does not chang the contents of the string value because strings are immutable.
  8. What does it mean if an object implements the IFormattable interface?
    • An object that implements the IFormattable interface contains a ToString method that accepts a format string and a format provider that can be used to provide culture specific information about the culture for which the string is being formatted.
    • Objects that implement IFormattable can be used with formatted output which contains formatting instructions that are given the ToString method.
  9. What does a CultureInfo object actually contain?
    • A CultureInfo instance contains information describing the formatting of output for use by a particular culture.
    • The CultureInfo instance contains references to the NumberFormatInfo and DateTimeFormatInfo object for the culture.
    • These specify such things as the ordering of dates when printed, the character to be used to mean currency and the separators between digits in large numbers.
    • A formatter can use those parts of a CultureInfo instances as it needs to produce output that reflects different regions of the world.
  10. Can you use string interpolation with strings of text that are created as the program runs?
    • String interpolation takes plce when the compiler works on literal string values in the program source.
    • A string that is idenified as an interpolated string by having a dollar character ($) added before the start of the string will be parsed and the names of the variables to be printed extracted.
    • This all happens at compile time, so it is not possible to use interpolation on string that are generated when the program runs.

1 Perform I/O Operations

Class hierarchies are an important element in the design of the data management mechanisms provided by .NET. The Stream class sets out the fundamental data movement commands and is used as the basis of classes that provide data storage on a variety of platforms. Streams can also be chained together so that character encoding, encryption and compression can be performed on data being moved to or from any stream-base store.

Here are some questions to consider:

  1. Why can't a program create and instance of the Stream type?
    • Because the Stream class is defined as abstract and intended to be used as a base class for child classes that contain implementions of the behaviours described by Stream.
  2. When would a program modify the file pointer in a file?
    • The file pointer value is maanged by a stream and specifies the position at which the next I/L transaction will be performed.
    • When a file is read, the pointer is at the beginning of the file.
    • The pointer can skip some of the file by updating the file pointer location.
    • Updating the value is faster than reading through the file.
    • When writing into a file, the point is updated as the file is written.
    • When a file is opened in append mode the file pointer is moved to the end of the file.
  3. What is the difference between UTF8 UNicode and UTF32 Unicode?
    • Unicode is a mapping between values and text characters.
    • The UTF8 standard maps Unicode characters onto one or more 8-bit storage locations.
    • The UTF32 standard maps Unicode characters onto one or more 32-bit storage locations.
  4. Is there such a thing as a "text file"?
    • The file system makes no distinciton between a "text" file and a "binary" file.
    • This distinction is made by programs using the file system to store files.
    • In C#, a text file is one that contains values which represent text.
    • The values in the text file will be encoded using a standard that maps numeric values onto characters. Typically Unicode.
  5. Can a program both read from and write to a single file?
    • Yes. If a stream is opened for ReadWrite access, then it can be read from and written to.
    • The file usually holds fixed length records, so a single record in the middle of a file can be changed without affecting any of the other records in the file.
  6. What is the difference between a TextWriter and a StreamWriter?
    • The TextWriter class is abstract and specifies operations that can be performed to write text into a stream.
    • The StreamWriter class is a child of the TextWriter class that can be instantiated and used to write text into a stream.
  7. Why is it useful for a stream class to have a constructor that accepts an object of the stream type?
    • A stream can be constructed from another stream to allow streams to be "chained" together.
    • The output of one stream can be sent into the next.
  8. What is the File class for?
    • It provides a set of very useful methods which allow the creation and population of files with only a small number of statements.
    • It also provides useful file management commands.
  9. Will you need to re-compile a program for it to be used on a system with a different file system?
    • No. The precise way a given file system works is completely hidden from the program using it.
  10. What happens to the files in a directory when you delete the directory?
    • An exception will be thrown when a program tries to delete a directory containing a file.
    • All the files in a directory must be deleted first.
  11. How can you tell if a file path is an absolute path to the file?
    • An absolute file path will start with the drive letter on which the file is stored and contains all the directories to be traversed to reach the file.
    • A relative path will not contain a drive letter.
  12. Why should you use the Path class to construct file paths in your programs?
    • The Path class provides a static Combine method that makes sure that all path separators are correctly inserted int the path that is created.
    • This is much less error prone that trying to make sure that the correct number of "\" characters have been included.
  13. What is the difference between HTTP and HTML?
    • HTTP (Hyper Text Transfer Protocol) defines the way that a client can give instructions to a web server.
    • HTMl (Hyper Text Markup Language) defines the format of documents that are sent back from the server in response to an HTTP request.
  14. When should you use the HttpWebRequest class to contact a web server?
    • The HttpWebRequest class is extremely customizable and provides much more flexibility that the WebClient class when creating requests to be sent to web servers.
    • Some forms of web request can only be sent using an HttpWebRequest.
  15. Can a program make web requests asynchronously?
    • Yes. The WebClient and HttpClient classes support the use of async and await making asynchronous web requests.
  16. Why is it a good idea to perform file operations asynchronously?
    • Because they are frequently the slowest actions performed on a computer.
    • Writing to a physical disk takes much longer than calculations, and disk transactions may take much longer if the system is busy.

2 Consume Data

Also, web services were covered; a means by which a server can expose resources in the form of method calls.

Here are some questions to consider:

  1. Does each user of a database need their own copy of the data?
    • A database server provides access to a central store of data.
    • It is a bad idea for each user to have their own copy of the data as it can become updated in different ways.
    • It is possible to keep a local copy of a database for use when the system is not connected to a network and then apply the updates to a database when the connection is available.
  2. What would happpen if two users of a database updated the same record at the same time?
    • A database server is designed to ensure that the stored data is always consistent.
    • If two users updated a particular record at the same time the updates wll be performed in sequence and the server will make sure the data isn't corrupted.
    • It is possible for a database user to flag actions as atomic so they are either completed or the database is "rolled back" to the state it had before the action.
  3. What would happen if two items in a database had the same ID value?
    • If the ID of an item is being used to uniquely identify the item, it is impossible to create two items with the same ID.
    • The database will generate ID values for items automatically when they are added to the database.
  4. Why do you need to protect your database connection string?
    • A database connection string is used to create a connection to the database.
    • If the database is not on the same computer as the program using it, the connection string will include the network address of the server and the authentication information.
    • Armed with this information anyone can create a connection to the database and send it SQL commands, wich would be a bad thing.
  5. Do you always have to write SQL queries to interact with a database?
    • SQL queries are the lowest level of communication with the databse.
    • ASP.NET applications can perform actions on objects in the program and then update the contents of the database with the new objects.
  6. What is an SQL injection attack, and how do you protect against one?
    • An SQL command is a construction that contains command and data elements.
    • An SQL injection attack is performed by "injecting" SQL commands into the data parts of the command.
    • For example, if a user is asked to enter their name, they can add malicious SQL commands to the name text.
    • The attacks are prevented using parameterized SQL commands, which specifically separate data from the command elements.
  7. You can see how to read JSON data from a server, but is it possible to store JSON formatted data values on a server?
    • If your application needs to be able to upload data to a server application it can use the REST (Representational State Transfer) model.
    • This makes use of the HTTP command set to allow an application to send data between the client and the server.
    • This data can be XML or JSON documents.
  8. What is the difference between a web service and just downloading JSON documents from a server?
    • A web service provides the client applicaiton with a "proxy object" that represents a connection to the service.
    • The client can call methods on the proxy to interact with the service.
    • This provides a much richer set of options than simply downloading a file.

    • It refers to the way that a data query can be expressed using language elements in the format called "query comprehension".
    • The LINQ operators allow the programmer to express thei intent directly without having to make a chain of method calles to create a query.
    • The compiler converts "query comprehension" notation queries into chains of method calls.
    • Programmers can create "method-based" queries if they prefer.
  2. Does LINQ add new features for data manipulation?
    • No. LINQ doesn't create any new data manipulation features, it just makes it easier for programmers to express their intent.
  3. Is it more efficient to build and perform our own SQL queries than to use LINQ?
    • Building your own SQl queries is slightly more efficient than using LINQ.
    • The very first time a LINQ query is used it will be compiled into the method sequence that is then called to deliver the result.
    • This is rearely a problem from a performance point of view, however, LINQ makes programmers more efficient.
  4. Why would the statement var counter; cause a compilation error?
    • When declaring a var type variable the compuler needs to be able to infer the type from the value being assigned to it.
    • If a declaration does not assign an initial value to the variable the compiler will refuse to compile the statement because it has no value to infer the type from.
  5. What is an anonymous type?
    • It is a type that has no name.
    • Normally a type is defined and then the new keyword is used to make new instances of that type.
    • With anonymous types, the instance is created without the associated type.
    • With LINQ, these objects can be returned as the results of queries. These objects contain custom result types that exactly match the requested data.
    • The objects must be referred to using the var keyword.
  6. What does "deferred execution" of a LINQ query mean?
    • A LINQ query will return a result that describes an iteration that can then be processed using a foreach construction.
    • If the query contains many result values, it takes a long time (and a lot of memory) to actually generate that result when the query is performed.
    • So instead, the iteration is evaluated and each result is generated in turn when it is to be consumed.
  7. What does the group behaviour do in a LINQ query?
    • It creates a new dataset grouped around the value of one item in the input dataset.
    • Aggregate operations can then be used on the group to do things such as count the number of tracks and sum the total length of tracks for each artist.
  8. What do the take and skip behaviours do?
    • The take behaviour creates a LINQ query that takes a particular number of items from the source set.
    • The skip behaviour skips down the dataset by a given number of items before the query starts taking values.
    • Used in combination they allow a programmer to work through a dataset one "page" at a time.
  9. What is the difference between query comprehension and method-based LINQ queries?
    • Both can be used interchangeably in programs.
    • You can use both in a given program, depending on which is most convenient at any given point in the program.
  10. What is the difference between the XDocument and XElement types?
    • The XDocument class can hold an XML document, including the elements that can be used to express a "fully formed" XML document, including metadata about the document contents.
    • An XElement can contain an element in an XML document, which can contain a tree of other XElement objects.
    • An XDocument contains XElement objects that contain the data in the document.
  11. What is the difference between the XmlDocument and XDocument types?
    • The XmlDocument implements a DOM (Document Object Model) for XML documents.
    • The XDocument builds on the ability of an XmlDocument to allow it to work with LINQ.
    • The same relationship applies between XmlElement and XElement.

4 Serialize and Deserialize Data

Serialization is very useful when you want to store or transfer structured data. It is susceptible to problems if the content or arrangement of data items that have been serialized change in later versions of an application. But, it is possible with sensible data design.

Here are some questions to consider:

  1. When should you use serialization in your program?
    • It's very useful when storing small amounts of structured data.
    • The highscore table for a game can be stored as a serialized object.
    • It would be less sensible to store large data strcutures as serialized objects, particularly if the object has to be repeatedly updated.
  2. Do you need a copy of the type to read a serialized object?
    • Serialization takes a snapshot of the data elemnts in a particular type.
    • During the deserialization process a new instance of the serialized type is created. This process requires the type to be deserialzied to be available on the receiving machine.
    • With binary serialization, the binary file contains type information that is compared with type information in the destination class. If this information doesn't match, the deserialization fails.
    • In XML and JSON serialization, this matching does not take place. These types can be regarded as more portable, at the expense of security.
    • The DataContract serialization process allows the serialized object to contain type information that can be checked during deserialization, but the format of the serialized file is still human readable XML.
  3. Can any data type be serialized?
    • Binary serialization can store any type of data, but XML serialization is restricted to numeric and text types.
  4. Can value types and reference types be stored using serialization?
    • Binary serialization can store reference types as references.
    • XML, JSON and DataContract serialization will resolve references to obtain values that are then serialized.
  5. Does serialization store all of the elements of a class?
    • Serialization does not store the methods in a class, or any static elements.
    • Binary serialization will store private data memebers of a type.
    • DataContract Serialization can store private data members as XML text.
  6. When should you use binary serialization and when should you use XML serialization?
    • Binary serialization is very useful for taking a complete snapshot of the data content of a class.
    • It forces the serialize and deserialize process to make use of identical classes.
    • It is very useful for transferring an object from one process to another, where the serialized data stream will not be persisted.
    • I'm not keen on using it to persist data for long periods of time because it is vulnerable to changes in the classes used.
    • Text serialization such as XMl is very sueful if you want to trasnfer data from one language or host to another. It is highly portable and useful for storing small amounts of structured data.
  7. Why should you be concerned about security when using binary serialization?
    • Binary serialization produces a stream that represents the entire contents of a class, including all private content.
    • With an understanding of the content, it is posible that it can be compromised, allowing the private content of an object to be viewed and changed.
    • Just because you can't view the contents of a binary serialized object with a text editor does not mean that it is immune to tampering.
  8. Can you use a custom serializer when serializing to an XML document?
    • A custom serializer allows the programmer to get control of the serializaiton process either by creating their own process or by getting control during the phases of the process.
    • These customization are only possible when using binary data serialization.
  9. Is it possible to encrypt a serialized class?
    • All serialziers use data streams to transfer data being serialized and deserialized.
    • It is possible to use an encrypting stream to compress serialized data.
  10. What is the difference between an XML serializer and a DataContract serializer?
    • Both XMl and DataContract serializers produce XML output.
    • With XML, if a class is marked as serializable, all of the data elements in the class will be serialized and the programmer must mark as NonSerialized any data members that should not be serialized.
    • In the case of DataContract serialization, the programmer must mark elements to be serialized.
    • The other function differnece is that private data memebers can be serialized using DataContract serialization.
